# CLAUDE.md (User-level)

このファイルは全プロジェクトに適用されるユーザーレベルの設定です。

## 言語設定

- 日本語でコミュニケーションする
- コードコメントやドキュメントも日本語で記述する

## コーディングスタイル

- コミットメッセージ: Conventional Commits形式のPrefixを付け、日本語で「なぜ」を重視
  - 例: `fix: 〇〇を修正`, `feat: 〇〇を追加`, `refactor: 〇〇を整理`

## サブエージェントのモデル指定

Agentツールでsubagentを起動する際、タスクの性質に応じてモデルを指定する:

- opus: 設計判断、アーキテクチャ分析、要件解釈、複雑なデバッグ（think系スキル）
- sonnet: コードレビュー、定型実装、テスト生成、品質チェック、コミット、PR作成（do系・auto系スキル）
- 判断基準: 「正確な推論が必要」ならopus、「手順に従って実行」ならsonnet

## 環境

- macOS / Homebrew / mise / fish

## 記法ルール

- 太字(**text**)を使わず、読みやすい文章を書く

## 対話ルール

- 不明点がなくなるまで繰り返し質問すること
- 選択肢は2〜4個に絞り、明確な説明を日本語で付けること

## セッション管理

- 承認済みの計画がある場合、各フェーズ完了後はユーザーに次のフェーズへ進むか確認すること
- ただしodinワークフローでは自律進行ルールに従う: Phase 3（理解要約）・Phase 4（計画）は承認不要で自動進行し、レビューチェーン（auto-peer-review / talk-review + Codex）で品質を担保する（odin/SKILL.md参照）
- セッションの終了はユーザーが判断する。「おやすみなさい」「終わりにしますか？」等の提案は禁止

## クリップボード連携

ドキュメントや成果物を新規作成した場合、報告時にファイルの絶対パスを `pbcopy` でクリップボードにコピーすること。
ユーザーがエディタやブラウザですぐ開けるようにするための仕組み。

対象:
- `.claude/artifacts/` 配下の成果物（設計書、要件定義、調査レポート、ADR、テスト計画等）
- 生成したHTMLファイル（プレゼン、デザインガイド、ビジュアルガイド等）
- デザインシステム関連ファイル（tokens.json, design.md等）
- その他、ユーザーに確認してもらうべき長文ドキュメントの新規作成

対象外: コードファイルの編集、設定ファイルの変更、小規模な修正

実行方法:
```bash
echo -n "/absolute/path/to/file" | pbcopy
```

## 長期記憶 (yb-memory)

過去のセッションの会話を検索できるツール `yb-memory` が利用可能です。

- 過去に解決した問題や議論した設計判断を参照したい場合:
  ```
  yb-memory search "検索クエリ" --limit 5
  ```
- 全プロジェクト横断検索:
  ```
  yb-memory search "検索クエリ" --all-projects
  ```

保存は自動的に行われます（セッション終了時のStopフック）。

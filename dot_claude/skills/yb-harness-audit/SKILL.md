外部スクリプトは使用しない。Claude自身がリポジトリの.claude/ディレクトリを分析してスコアリングする。

## 分析対象カテゴリ（各0-10点、合計0-70点）

### 1. Skills (0-10)

.claude/skills/ 配下のSKILL.mdを分析する。

- SKILL.mdの数
- フォーマット準拠（frontmatter有無）
- トリガー条件の明確さ

スコア基準:
- 0: なし
- 3: 1-2個
- 5: 3-5個
- 7: 6-10個
- 10: 10+個かつフォーマット統一

### 2. Agents (0-10)

.claude/agents/ 配下のエージェント定義を分析する。

- エージェント定義数
- model指定の有無
- tools制限の適切さ

スコア基準:
- 0: なし
- 5: 1-2個
- 8: 3+個かつmodel指定あり
- 10: 役割分離が明確

### 3. Hooks (0-10)

settings.json のhooks設定を分析する。

- hooks設定数
- イベントカバレッジ（PreToolUse/PostToolUse/Stop等）

スコア基準:
- 0: なし
- 3: 1イベント
- 5: 2イベント
- 8: 3+イベント
- 10: エラーハンドリング含む

### 4. Rules (0-10)

.claude/rules/ 配下のルールファイルを分析する。

- ルールファイル数
- カバレッジ（セキュリティ、テスト、パフォーマンス等）

スコア基準:
- 0: なし
- 5: 1-2個
- 8: 3+個
- 10: セキュリティ+テスト+パフォーマンス揃い

### 5. Memory (0-10)

CLAUDE.md の充実度を分析する。

- 文字数
- セクション数
- 具体性

スコア基準:
- 0: なし
- 3: 最小限
- 5: 基本情報
- 8: 詳細なルール
- 10: 環境・スタイル・ワークフロー網羅

### 6. Permissions (0-10)

settings.json のpermissions.allow設定を分析する。

- 許可コマンドの網羅性
- セキュリティの適切さ

スコア基準:
- 0: なし
- 5: 基本コマンド
- 8: 読み取り系+git系
- 10: 網羅的かつセキュアなリスト

### 7. MCP (0-10)

MCPサーバー設定を分析する。

- MCPサーバー設定の有無
- 活用度

スコア基準:
- 0: なし
- 5: 1-2個
- 8: 3+個
- 10: プロジェクトに最適化

## 出力形式

以下の形式で結果を出力する:

```
カテゴリ        スコア  評価
Skills          8/10   良好
Agents          6/10   改善余地あり
Hooks           3/10   最低限
Rules           0/10   未設定
Memory          8/10   良好
Permissions     5/10   基本のみ
MCP             5/10   基本のみ
合計            35/70

改善提案:
1. [優先度高] ...
2. [優先度中] ...
3. [優先度低] ...
```

改善提案は優先度順に並べ、具体的なアクション（何を追加・変更すべきか）を記載する。

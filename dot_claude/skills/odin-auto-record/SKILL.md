# odin-auto-record

開発中に気づいた改善点・学び・教訓をinsightsファイルに蓄積するスキル。
odin-auto-improveのステップ0で自動参照され、改善サイクルのフィードバックループを形成する。

## 背景

odin-auto-improveはWebSearch+メタ認知で自己分析するが、実運用での気づきは開発者しか持っていない。
このスキルで「現場の声」をinsightsファイルに蓄積することで、改善の質が向上する。

## コンテキスト検出

### odinコンテキストがある場合
- odinから呼び出されている。記録内容はodinコンテキストから取得する。

### odinコンテキストがない場合
- ユーザーが直接呼び出している。ステップ1でヒアリングする。

## ステップ1: 記録内容のヒアリング

ユーザーに以下を確認する（AskUserQuestion）:

1. 何について記録するか（自由記述、または会話の要約から抽出）
2. タグの選択（複数選択可）:
   - [スキル品質]: odinスキル群の構造・指示・振る舞いに関する気づき
   - [進め方]: ワークフロー・プロセスに関する気づき
   - [コミュニケーション]: ユーザーとの対話・説明に関する気づき
   - [ツール選択]: ツールの使い方・選択に関する気づき
   - [実装判断]: コード・設計の判断に関する気づき
   - [学び]: 技術的な学び・ベストプラクティスの発見

ユーザーが「さっきのやつ記録して」のように曖昧に言った場合は、直近の会話から改善点を推測してAskUserQuestionで確認する。

#### 完了チェックポイント（ステップ1）
- 記録内容とタグが確定していること

## ステップ2: insightsファイルへの記録

1. `.claude/artifacts/insights-{yyyyMMdd}.md` の存在を確認する
   - 存在する場合: 末尾にエントリを追記する
   - 存在しない場合: 新規作成する
2. 以下の形式でエントリを追加する:

```markdown
### {HH:mm} [{タグ}] {要約タイトル}

{詳細な記述}

- 状況: {どのような作業中に気づいたか}
- 改善案: {具体的にどうすべきか}
- 対象: {改善対象のスキル名やファイル名。不明なら「未特定」}
```

3. 同日の既存エントリと重複していないか確認する（重複する場合は既存エントリに補足を追記する）

#### 完了チェックポイント（ステップ2）
- insightsファイルにエントリが追加されていること
- タグ・要約・詳細が揃っていること

## ステップ3: 記録完了の報告

記録した内容をユーザーに報告する:
- ファイルパス
- 記録したエントリの要約
- 「次回odin-auto-improve実行時に自動参照されます」と伝える

#### 完了チェックポイント（ステップ3）
- ユーザーに記録完了が報告されていること

## odin-auto-improveとの連携

odin-auto-improveのステップ0で以下のように参照される:
- `.claude/artifacts/insights-*.md` を時系列で読み込む
- タグでフィルタリング:
  - [スキル品質]: 常に取り込む
  - [進め方]: スキル構造への示唆がある場合のみ
  - [コミュニケーション]: スキルの指示文に関する示唆がある場合のみ
  - [ツール選択]: allowed-toolsや手順に関する示唆がある場合のみ
  - [実装判断] [学び]: 取り込まない
- 取り込み済みエントリはエピソディックメモリに記録され、次回以降スキップされる

## Examples

### Example 1: CodeRabbit指摘対応での気づきを記録

ユーザー: 「CodeRabbitの指摘対応で気づいたんだけど、修正したら自動でコメント返信してほしい。記録して」

```
ステップ1: ヒアリング
  内容: pr-5-coderabbit-fixに修正後の自動返信機能がない
  タグ: [スキル品質]
  → ユーザー確認OK

ステップ2: insightsファイルへの記録
  → .claude/artifacts/insights-20260328.md に追記
  エントリ: "14:30 [スキル品質] pr-5-coderabbit-fixに修正後の自動コメント返信機能を追加すべき"

ステップ3: 記録完了報告
  → 「記録しました。次回odin-auto-improve実行時に自動参照されます」
```

### Example 2: ワークフローの改善点を記録

ユーザー: 「さっきのPRレビューで気づいたけど、auto-reviewとtalk-reviewの使い分けが分かりにくい。メモして」

```
ステップ1: ヒアリング
  内容: auto-reviewとtalk-reviewの使い分け基準が不明確
  タグ: [スキル品質], [コミュニケーション]
  → ユーザー確認OK

ステップ2: insightsファイルへの記録
  → .claude/artifacts/insights-20260328.md に追記

ステップ3: 記録完了報告
```

## 対応シーン一覧

odinはあらゆる開発シーンに対応する。以下は代表的なシーンとスキルの組み合わせである。

| シーン | 典型的な入力 | 使用スキル（実行順） |
|--------|------------|-------------------|
| 新規機能開発 | 「通知機能を追加したい」 | research → requirements → design → plan → implement → test → commit → pr |
| 不具合修正 | 「ログイン画面でエラーが出る」 | investigate → implement → test → commit → pr |
| 要件定義支援 | 「要件を整理したい」 | research → clarify → requirements |
| リファクタリング | 「認証周りのコードを整理したい」 | research → analyze → plan → refactor → commit → pr |
| PRレビュー・マージ | 「PR #42をレビューしてマージして」 | review → merge |
| 見積もり依頼 | 「この機能どれくらいかかる？」 | research → requirements → plan（見積もりのみ） |
| アーキテクチャ理解 | 「このプロジェクトの構成を教えて」 | research → explain |
| 品質チェック | 「品質チェックして」 | auto-quality |
| 大量タスク並列処理 | 「5画面を一気に実装して」 | plan → implement(並列) → test → commit → pr |
| 技術選定・比較 | 「状態管理ライブラリを比較したい」 | research → analyze → propose |
| プロダクト方針策定 | 「次のスプリントの方針を決めたい」 | research → clarify → propose |
| バグ調査レポート | 「この問題を企画チームに説明して」 | investigate → explain（企画PO向け） |
| テスト強化 | 「カバレッジを上げたい」 | analyze → test → commit |
| PR作成 | 「PRを作って」 | commit → pr |
| コード説明 | 「この関数の動きを新メンバーに説明して」 | explain（新メンバー向け） |
| UIデザイン実装 | 「ランディングページのデザインを作りたい」 | ux-3-core → ux-2-tokens → research → design → ux-4-styling → implement → test → commit → pr |
| デザイン参照・トークン抽出 | 「このサイトのデザインを参考にしたい」 | ux-3-core → ux-2-tokens（逆抽出） → design |
| コンポーネントデザイン | 「カードコンポーネントをデザインして」 | ux-3-core → ux-4-styling → implement → test → commit → pr |
| 学習セッション | 「学習して」「今日の学習」 | learn（学習セッション） |
| 教育付きPRレビュー | 「教育モードでレビューして」 | learn-review（レビュー+教育対話） |
| フラッシュカード復習 | 「復習して」 | learn（復習モード） |
| スキルマップ確認 | 「スキルマップを見せて」 | learn（進捗レポート） |
| 技術質問 | 「なぜ〜」「〜の違いは」「〜はどう動く」 | learn（ソクラテス式回答 + フラッシュカード生成） |
| 改善点・学びの記録 | 「記録して」「メモして」「insightを残して」 | auto-record |
| CodeRabbitレビュー対応 | 「CodeRabbitの指摘を直して」「CRの指摘を対応して」 | pr-5-coderabbit-fix |
| Web検索・技術調査 | 「〇〇について調べて」「最新の〇〇を検索して」 | codex-search（Codex環境ではCodex委任、それ以外はWebSearch） |

### シーン判定ルール

ユーザー入力から以下のキーワード・パターンでシーンを自動判定する:

- 「作りたい」「追加したい」「実装して」→ 新規機能開発
- 「エラー」「バグ」「動かない」「落ちる」→ 不具合修正
- 「要件」「仕様」「ヒアリング」→ 要件定義支援
- 「リファクタ」「整理」「きれいに」→ リファクタリング
- 「レビュー」「PR」→ PRレビュー・マージ
- 「見積もり」「工数」「どれくらい」→ 見積もり依頼
- 「教えて」「説明して」「構成」→ アーキテクチャ理解
- 「品質」「lint」「テスト」→ 品質チェック
- 「比較」「選定」「どっちがいい」→ 技術選定・比較
- 「コミット」→ PR作成
- 「デザイン」「スタイル」「UI」「見た目」「レイアウト」→ UIデザイン実装
- 「参考にしたい」「このサイト」「トークン抽出」→ デザイン参照・トークン抽出
- 「コンポーネント」「ボタン」「カード」「モーダル」→ コンポーネントデザイン
- 「学習」「勉強」「今日の学習」「スキルマップ」→ 学習セッション
- 「教育モード」「レビューで学びたい」→ 教育付きPRレビュー
- 「復習」「フラッシュカード」→ フラッシュカード復習
- 「なぜ」「違いは」「どう動く」「仕組み」「〜とは」→ 技術質問
- 「記録して」「メモして」「insightを残して」→ 改善点・学びの記録
- 「CodeRabbitの指摘を直して」「CRの指摘を対応して」「レビューコメントを修正して」→ CodeRabbitレビュー対応
- 「調べて」「検索して」「最新の〇〇」「〇〇について調査」→ Web検索・技術調査

シーンが不明確な場合はPhase 2で確認する。複数シーンに該当する場合は最も包括的なシーンを選択する。

## Examples

### 例1: 初回セットアップ

ユーザー: 「/odin 学習して」

```
→ skillmap-latest.md が存在しない
→ ステップ A: スキルマップ初期化
→ 自己評価ヒアリング
→ 初期レベル設定
→ ステップ D: 最初の学習セッション開始
  → 目標との差が最大の領域を推奨
```

### 例2: 復習セッション

ユーザー: 「/odin 復習して」

```
→ ステップ C: フラッシュカード復習
→ 期限切れカード5枚を復習
→ 正答率 3/5 (60%)
→ 間違えた2枚の次回復習を+1日にリセット
→ 正解した3枚の次回復習間隔を延長
```

### 例3: 特定トピックの学習

ユーザー: 「/odin CSSのコンテナクエリについて学びたい」

```
→ ステップ D: トピック指定あり → CSS設計
→ WebSearchで最新のコンテナクエリのベストプラクティスを調査
→ 現在レベル: Understand → Apply を目指す
→ コンテナクエリの基本構文 → 実践課題 → ソクラテス式Q&A
→ フラッシュカード3枚生成
→ スキルマップ更新
```

### 例4: 進捗確認

ユーザー: 「/odin スキルマップを見せて」

```
→ ステップ B: 進捗レポート
→ 総合進捗: 23%
→ 成長が速い領域: FEアーキテクチャ（Understand→Apply）
→ 要注意: CSS設計（復習7日超過）
→ 推奨: 1. CSS設計（復習期限超過）、2. アクセシビリティ（未着手）
```

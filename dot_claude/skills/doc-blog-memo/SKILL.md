# ブログ記事メモ生成

会話の中で調査・解決した内容をもとに、ブログ記事の骨組み（アウトライン）をMarkdown形式でチャット上に出力する。
ファイルは作成せず、テキスト出力のみ行う。

## 出力ルール

- 日本語で出力する
- 太字（`**text**`）は使わない
- 完全な文章ではなく、要点を箇条書きにした骨組みを出力する
- セクション内の説明は箇条書き（`-`）で記述する
- コードやコマンドは適宜インラインコードやコードブロックで示す

## 出力構造

以下の構造でMarkdownテキストを出力する:

```
# タイトル（問題と解決策が一目でわかるもの）

## セクション1
- 箇条書きで要点

## セクション2
- 箇条書きで要点

...

## まとめ
- 記事全体のポイントを箇条書きで整理
```

## セクション構成の考え方

内容に応じて適切なセクションを選ぶ。以下は典型的なパターンだが、機械的に全てを使う必要はない。記事の内容に合わせて取捨選択・追加する。

- 発生した問題: 何が起きたか、どんな症状だったか
- 環境: OS、ツール、バージョンなど再現に必要な情報
- 原因: なぜ問題が起きたのか、調査で判明したこと
- 背景知識: 問題を理解するために必要な前提知識や仕組みの説明
- 対策: 具体的な解決方法、コマンド、設定変更
- まとめ: 記事全体のポイントを整理（必ず末尾に配置する）

## 情報源

会話の中で実際に行った以下の内容から情報を抽出する:

- 実行したコマンドとその結果
- 調査で判明した事実
- ユーザーとのやり取りで得られた文脈や補足情報
- 解決に至った手順

会話に含まれない情報を推測で補わない。

# 2025-2026 Webデザイントレンド照合基準

design-dissect スキルがトレンド照合時に参照するデータ。
分析対象サイトが「トレンドに沿っている / 独自路線 / クラシック」を判定する基準。

最終更新: 2026-04

---

## 1. レイアウト

### トレンド

- ベントグリッド（Bento Grid）: 大小異なるカードのモジュラーグリッド。SaaS上位100サイトの67%採用。2026年は角丸誇張（12-24px）+マイクロインタラクション付きの「Bento 2.0」が主流
- スクロールテリング: パララックス+スクロールアニメーションを組み合わせたナラティブ体験
- 非対称・アンチグリッド: 有機的な形状、意図的な非対称性で「人間的」なインターフェースを志向
- ヒーローセクション: フルスクリーンビデオループ / テキストオンリー（大型タイポ） / エディトリアルグリッド（雑誌的構成）

### 判定基準

| 判定 | 兆候 |
|------|------|
| トレンド | ベントグリッド、非対称レイアウト、スクロールトリガーアニメーション、CSS Gridの活用 |
| 独自路線 | 意図的に従来パターンを崩している（ネオブルータリズム等） |
| クラシック | 従来型12カラムグリッド、静的ページ構成、パララックスなし |

---

## 2. タイポグラフィ

### トレンド

- 大型タイポグラフィ: ヒーローセクションでオーバーサイズ文字。ブランド個性を即座に伝達
- 可変フォント: スクロール/ホバーに応じてウェイト・幅がリアルタイムに変化。レスポンシブとの親和性が高い
- セリフ体の復権: AI生成の均質感へのカウンター。信頼感+スタイリッシュさの両立。現代的セリフ（シャープ、高コントラスト、太めシェイプ）
- キネティックタイポグラフィ: スクロールで「呼吸する」見出し、ホバーで拡張する文字

### 判定基準

| 判定 | 兆候 |
|------|------|
| トレンド | 可変フォント使用、大型タイポ（見出し48px以上）、セリフ体の見出し、キネティック効果 |
| 独自路線 | カスタムフォント、手書き風、実験的タイポグラフィ |
| クラシック | サンセリフ一辺倒、固定サイズ、小さめの見出し |

---

## 3. カラー

### トレンド

- ダークファースト設計: トグルではなくダークモードを正当なデザインアプローチとして採用。SaaS・開発者ツール・メディアサイトで特に採用
- 二極化: ミュート/ニュートラル（トープ、ベージュ、パステル） vs ビビッド/ボールド（ドーパミンカラー、ネオングラデーション）
- ソフトグラデーション: レインボーではなく、ソフトなオンブレ遷移やデュアルトーンの洗練されたグラデーション
- Pantone 2026 Color of the Year: Cloud Dancer（PANTONE 11-4201、ニュートラルカラー）

### 判定基準

| 判定 | 兆候 |
|------|------|
| トレンド | ダークファースト、ミュートトーン+ビビッドアクセント、ソフトグラデーション、OKLCHカラースペース使用 |
| 独自路線 | 強い単色使い、ブランド固有の配色体系、意図的なモノクロ |
| クラシック | 白背景+ブランドカラーのみ、レインボーグラデーション、Material Design標準色 |

---

## 4. インタラクション・アニメーション

### トレンド

- 意味のあるマイクロインタラクション: 装飾ではなく「確認・発見・安心」のため。ホバー状態、ボタンフィードバック、状態変化のアニメーション
- スクロールトリガーアニメーション: GSAP、Framer Motionなどの軽量JSライブラリで実装。「装飾のための動き」から「意味を伝える動き」へ
- Spatial UX（空間的UX）: ボタンがオブジェクトのように振る舞い、コンポーネントが重さと意図を持って応答。動的な奥行き、レイヤードパララックス
- アダプティブモーション: ユーザー行動に応じてアニメーション自動調整。prefers-reduced-motion対応
- 3D要素: Spline, Unicorn Studio等で一般デザイナーにも普及

### 判定基準

| 判定 | 兆候 |
|------|------|
| トレンド | 意味のあるマイクロインタラクション、GSAP/Framer Motion使用、スクロールトリガー、prefers-reduced-motion対応 |
| 独自路線 | WebGL重視、実験的インタラクション、独自アニメーションライブラリ |
| クラシック | 装飾的フェードイン、jQuery animate、アニメーションなし |

---

## 5. UIコンポーネント

### トレンド

- グラスモーフィズム進化形: 初期の過剰なブラーから進化。繊細な半透明レイヤーで本物の奥行き。コントラスト・アクセシビリティ改善
- ネオブルータリズム: 太い輪郭線、激しいボーダー、クラッシュカラー、モノスペースフォント。軽量・高速。若さと真正性を表現
- AIネイティブUI: Generative UI（AIがUIを動的生成）、OUI（Orchestrated User Interface）、チャットインターフェース（ChatKit, CopilotKit）
- マルチモーダルインタラクション: 音声・ジェスチャー・ビジョンを統合

### 判定基準

| 判定 | 兆候 |
|------|------|
| トレンド | グラスモーフィズム（繊細な使い方）、ネオブルータリズム、AIチャットUI、Generative UI |
| 独自路線 | 独自のコンポーネントスタイル、アート系の表現 |
| クラシック | フラットデザイン、Material Design準拠のみ、Bootstrap標準 |

---

## 6. デザインシステム

### トレンド

- AI最適化DS: 静的なライブラリから「生きた生成エンジン」へ。CLAUDE.md + tokens.json + 禁止パターンの3点セット
- DTCGトークン標準: Design Tokens Community Group標準に準拠。ツール間の相互運用性を確保
- セマンティックコンポーネント: 見た目ベース（primary/secondary）から意図ベース（アクションティア、リスクレベル）の分類へ
- MCP対応ドキュメント: 機械可読インテリジェンスハブとしてのドキュメント。AIエージェントがDSを理解・活用
- design.md: デザインルールをMarkdownで一元管理。AI生成の一貫性を保証

### 判定基準

| 判定 | 兆候 |
|------|------|
| トレンド | DTCGトークン使用、AI最適化（CLAUDE.md等）、MCP対応、セマンティックコンポーネント |
| 独自路線 | 独自のトークン体系、カスタムDSツール |
| クラシック | 静的Figmaライブラリのみ、DSなし、CSSフレームワーク依存 |

# デザイン分析フレームワーク（5軸）

design-dissect スキルが分析時に参照するフレームワーク。
5つの軸で「なぜこのデザインが効いているのか」を構造的に説明する。

---

## 軸1: 視覚原則

### ゲシュタルト6原則チェックリスト

| 原則 | 説明 | チェック項目 |
|------|------|------------|
| 近接 | 近くにある要素は同じグループと知覚される | 関連する情報が空間的に近接しているか。カード内の要素は密集、カード間は離れているか |
| 類似 | 色・形・サイズが似た要素は同類と知覚される | 同じ役割の要素が視覚的に統一されているか（色、形、サイズ） |
| 閉合 | 不完全な形でも脳が補完して全体像を認識する | 枠線の省略やオープンなレイアウトでもグループが認識できるか |
| 連続 | 視線は滑らかな線や曲線に沿って流れる | ステップインジケータやプログレスバーで視線を誘導しているか |
| 共同運命 | 同じ方向に動く要素は同グループと認識される | ホバー時に関連要素が同時にアニメーションするか |
| 図と地 | 前景と背景を区別して知覚する | モーダルの背景暗転、カードの影で前景/背景が明確か |

定性チェック: ブラーテスト（ページを50%ぼかして主要な情報群が識別できるか）

### 視覚階層の6構成要素

| 要素 | 効果 | チェック項目 |
|------|------|------------|
| サイズ | 大きい要素ほど先に注目される | 見出し > 小見出し > 本文 の順にサイズが下がっているか |
| 色とコントラスト | 鮮やかな色・高コントラストが目立つ | CTAボタンが周囲と十分なコントラストを持っているか |
| 位置 | ページ上部・左上が最も注目度が高い | 最重要情報がF字型/Z字型の視線パターンに沿っているか |
| タイポグラフィ | フォントウェイトの差で重要度を表現 | ウェイトの差が明確に階層を作っているか |
| 余白 | 要素の周囲に余白を取ると孤立して目立つ | 重要な要素の周囲に十分な余白（隔離効果）があるか |
| テクスチャ・装飾 | シャドウ、ボーダー、背景色で層を表現 | 装飾が階層の表現に貢献しているか（過剰ではないか） |

定性チェック: 3秒テスト（ページを3秒見て最初に目に入る要素はビジネス上最も重要な要素か）

### グリッドシステム

| 種類 | 特徴 | 判定方法 |
|------|------|---------|
| カラムグリッド | 12カラムが標準。2,3,4,6分割が容易 | 開発者ツールでグリッドオーバーレイを確認 |
| モジュラーグリッド | カラムとロウの交差でモジュールを作る | ダッシュボードやカードレイアウトで使われる |
| ベースライングリッド | テキストの垂直方向の整列を制御 | 行間が一定間隔に揃っているか |
| ベントグリッド | 大小異なるカードのモジュラー構成 | SaaSで67%採用（2025-2026トレンド） |

## 軸2: タイポグラフィ

### タイプスケール比率表

| 比率名 | 倍率 | 用途 | 計算例（base 16px） |
|--------|------|------|-------------------|
| Minor Second | 1.067 | 変化が微小。限定的 | 16 → 17.1 → 18.2 |
| Major Second | 1.125 | モバイルUI、コンパクト | 16 → 18 → 20.3 |
| Minor Third | 1.200 | 汎用、ブログ | 16 → 19.2 → 23.0 |
| Major Third | 1.250 | 一般的なWebサイト（推奨出発点） | 16 → 20 → 25 → 31.3 |
| Perfect Fourth | 1.333 | 多様なコンテンツ（最も広く使われる） | 16 → 21.3 → 28.4 → 37.9 |
| Augmented Fourth | 1.414 | 強い階層感、LP向け | 16 → 22.6 → 32.0 |
| Perfect Fifth | 1.500 | 大画面向け | 16 → 24 → 36 → 54 |
| Golden Ratio | 1.618 | アート系、ビジュアル重視 | 16 → 25.9 → 41.9 |

検出方法: 抽出したフォントサイズをpx値でソートし、隣接サイズ間の比率を計算。最頻比率を上記に±0.03でマッチング。

### フォントペアリング原則

| アプローチ | 説明 | 例 | 評価 |
|-----------|------|------|------|
| 調和 | 同一ファミリーのウェイト違い | Inter Regular + Inter Bold | 安全、統一感 |
| 対比 | 異なる性格のフォントを組み合わせ | セリフ見出し + サンセリフ本文 | 視覚的興味、王道 |
| 衝突 | 似ているが微妙に違うフォント | 避けるべきパターン | NG |

チェック項目:
- x-height（小文字xの高さ）が近いフォント同士か
- フォント数は2〜3種に限定されているか
- 見出しと本文で役割が明確に分かれているか

### 行間・字間の基準値

| 項目 | 最適値 | 根拠 |
|------|--------|------|
| 行間（本文） | line-height 1.4〜1.6 | WCAG基準。1.5が最も読みやすい |
| 行間（見出し） | line-height 1.1〜1.3 | 見出しは本文より詰める |
| 字間 | フォントサイズの0.12倍以上 | WCAG 2.2 SC 1.4.12 |
| 1行文字数（英語） | 45〜75文字 | Baymard Institute研究 |
| 1行文字数（日本語） | 25〜40文字 | 目安値 |

## 軸3: 配色

### カラーハーモニー5分類

| ハーモニー | 色相環上の関係 | 効果 | 判定方法 |
|-----------|--------------|------|---------|
| 補色 | 色相差 150-210° | 高コントラスト、エネルギッシュ | primary-secondaryの色相差を計算 |
| 類似色 | 色相差 0-60° | 穏やか、調和的 | パレット内の色相が近接しているか |
| トライアド | 120°間隔の3色 | 活気があり多彩 | 3色の色相が均等配置か |
| スプリット補色 | 補色の両隣 | 補色より柔らかいコントラスト | 2色が補色の±30°付近か |
| モノクロマティック | 同一色相の明度・彩度違い | 統一感、洗練 | パレットが同一色相のみか |

### WCAGコントラスト比基準

| 対象 | 最低基準（AA） | 推奨基準（AAA） |
|------|--------------|---------------|
| 通常テキスト（<18px） | 4.5:1 | 7:1 |
| 大テキスト（≥18px or 14px太字） | 3:1 | 4.5:1 |
| UIコンポーネント・グラフィック | 3:1 | - |

コントラスト比の計算: 相対輝度 L = 0.2126R + 0.7152G + 0.0722B（sRGBリニア化後）

### 色の心理効果とブランド適合性

| 色 | 心理効果 | 適合する業界 |
|----|---------|------------|
| 青 | 信頼、能力、安定 | 金融、テクノロジー、医療 |
| 赤 | 興奮、エネルギー、緊急 | 飲食、エンタメ、セール |
| 緑 | 健康、自然、環境 | ヘルスケア、オーガニック、環境 |
| 黄 | 楽観、注意喚起 | 注意表示、子供向け |
| 紫 | 高級感、創造性 | 美容、ラグジュアリー |
| 黒 | 権威、洗練 | ラグジュアリーブランド |
| 白 | 清潔、シンプル | ミニマルデザイン全般 |

重要: 個別の色の意味より「ブランドの性格と色の適合性」が重要。

## 軸4: スペーシング

### 8pxグリッドシステム

| スケール | 値 | 用途 |
|---------|-----|------|
| 3xs | 2px | ごく微細な調整 |
| 2xs | 4px | アイコンとラベルの間 |
| xs | 8px | 密接に関連する要素間 |
| sm | 12px | リスト項目間 |
| md | 16px | コンポーネント内部パディング |
| lg | 24px | 要素グループ間 |
| xl | 32px | セクション内ブロック間 |
| 2xl | 48px | セクション間 |
| 3xl | 64px | 大きなコンテンツブロック間 |
| 4xl | 96px | ページセクション間 |

8pxが選ばれる理由: 2と4で割り切れる。Retinaディスプレイで偶数値がきれいにスケール。一般的な画面サイズ（360, 375, 390, 768, 1024, 1440等）と親和性が高い。

### スペーシング法則

| 法則 | 説明 | チェック項目 |
|------|------|------------|
| 内部 ≤ 外部 | コンポーネント内部のパディング < コンポーネント間のマージン | カード内paddingがカード間gapより小さいか |
| 親子関係 | 親コンテナのパディング > 子要素間のマージン | セクションpaddingがカード間gapより大きいか |
| 一貫性 | 同じ階層のコンポーネントには同じスペーシング | 同種のカード間スペーシングが統一されているか |
| リズム | 等間隔パターンで予測可能性を生む | セクション間余白が一定のリズムを持つか |
| リズムの崩し | 意図的な変更で注目を集める | CTAの前後に特に広い余白があるか |

8pxグリッド準拠率の計算: 抽出したスペーシング値のうち、8の倍数である値の使用回数 / 全スペーシング値の使用回数 × 100

## 軸5: UIパターン

### ナビゲーション4型

| 型 | 特徴 | 適するサイト |
|---|------|------------|
| トップナビ（水平） | 主要カテゴリが少ない。スキャンしやすい | コーポレート、LP、カテゴリ少のSaaS |
| ハンバーガーメニュー | モバイル標準だが発見性が低い | モバイルファースト、コンテンツ重視 |
| サイドナビ（垂直） | 多くのメニュー項目を収容。SaaSダッシュボード標準 | SaaS、管理画面、複雑なアプリ |
| メガメニュー | カテゴリの全体像を俯瞰できる | EC、大規模メディア、多カテゴリサイト |

### CTA分析

| チェック項目 | 基準 |
|------------|------|
| プライマリCTAのコントラスト | 周囲と明確なコントラスト差があるか |
| ファーストビュー内配置 | スクロールなしで最初のCTAが見えるか |
| 繰り返し配置 | 長いページでCTAが適切に繰り返されているか |
| プライマリ/セカンダリの区別 | ボタンの視覚的重みに明確な差があるか |
| ラベルの明確さ | 「今すぐ始める」等、アクションが明確か |

### Hookモデル（Nir Eyal）

| 段階 | 説明 | Web上の具体例 | チェック項目 |
|------|------|-------------|------------|
| トリガー（外部） | 行動のきっかけ | 通知バッジ、ポップアップ、バナー | 外部トリガーがどこに配置されているか |
| トリガー（内部） | 感情に基づくきっかけ | FOMO、退屈、不安 | 緊急性の演出（残り在庫、カウントダウン）の有無 |
| アクション | 報酬を期待して行う最小行動 | クリック、スクロール、スワイプ | 操作のハードルが低いか |
| 可変報酬 | 予測不能な報酬が習慣を形成 | フィード更新、いいね数、新着コンテンツ | 可変的な報酬要素があるか |
| 投資 | ユーザーが価値を蓄積する行為 | プロフィール作成、お気に入り登録 | 投資を促す仕組みがあるか |

### 業界別パターン

| 業界 | 代表的パターン |
|------|-------------|
| SaaS | ダッシュボード、オンボーディングフロー、空の状態、3カラム料金表、ナビサイドバー |
| EC | 商品カード（画像+名前+価格+評価）、左サイドフィルタ、カートバッジ、レビュー |
| メディア | カードレイアウト、無限スクロール/ページネーション、カテゴリタグ |
| コーポレート | ヒーローセクション、実績セクション、CTA繰り返し、問い合わせフォーム |
| ポートフォリオ | 大型ビジュアル、プロジェクトグリッド、ミニマルナビ |

## 実践的分析手順（9ステップ）

1. 第一印象テスト（3秒）: ページを見て最初に何に目が行くか記録 → ビジネス上最重要な要素と一致するか
2. ブラーテスト: スクリーンショットにぼかしを掛け、視覚階層の有効性を確認
3. グリッド確認: 要素が暗黙のグリッドに沿って配置されているか
4. タイプスケール計測: 各テキスト要素のフォントサイズを計測し、比率の一貫性を確認
5. 配色抽出: カラーパレットを抽出し、ハーモニーの法則との整合性を確認
6. スペーシング計測: マージン/パディングを計測し、8pxグリッドへの準拠を確認
7. コントラスト比チェック: テキストと背景のコントラスト比をWCAG基準で検証
8. UIパターン分類: 使用されているナビゲーション・CTA・フォームのパターンを特定
9. フック分析: Hookモデルの4段階でトリガー・報酬・投資の仕組みを特定

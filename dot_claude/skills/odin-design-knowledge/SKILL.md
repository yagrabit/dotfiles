# design-knowledge

design-dissect で蓄積したデザイン分析ナレッジを検索・比較・統計するスキル。

## 言語

- ユーザーとのやりとりは日本語で行う

## データソース

- 分析レポート: `.claude/artifacts/dissect-*.md`（learnモード出力）
- 監査レポート: `.claude/artifacts/audit-*.md`（auditモード出力）
- インデックス: `.claude/artifacts/design-knowledge-index.md`

## 機能判定

ユーザーの入力から以下の4機能を判定する。曖昧な場合はAskUserQuestionで確認する。

| キーワード | 機能 |
|-----------|------|
| 「検索」「探して」「〇〇なサイト」 | ナレッジ検索 |
| 「比較」「違い」「AとB」 | サイト横断比較 |
| 「統計」「よく使われている」「傾向」 | パターン統計 |
| 「進捗」「何サイト」「学習状況」 | 学習進捗 |

---

## 機能1: ナレッジ検索

ユーザーのクエリで分析レポートを全文検索し、該当するレポートを提示する。

手順:
1. Grep で `.claude/artifacts/dissect-*.md` と `audit-*.md` を対象に検索
2. ヒットしたレポートの「サイト概要」「学びのポイント」セクションを抽出
3. 検索結果を一覧で提示

例: 「青系のカラーパレットを使っているサイト」
→ カラーパレットセクションで青系（色相200-250°）のprimaryを持つレポートを検索

## 機能2: サイト横断比較

2つのレポートから指定セクションを抽出し、比較表を生成する。

手順:
1. ユーザーが指定した2サイト（または最新2件）のレポートを読み込む
2. 指定された軸（タイポグラフィ、カラー等）のセクションを抽出
3. 比較表を生成して提示

出力形式:
```
## {サイトA} vs {サイトB}: {軸名}比較

| 項目 | {サイトA} | {サイトB} | 評価 |
|------|----------|----------|------|
```

比較可能な軸:
- カラーパレット（色数、ハーモニー、コントラスト比）
- タイポグラフィ（スケール比率、フォント種類、行間）
- スペーシング（ベース単位、8px準拠率）
- UIパターン（ナビゲーション型、CTA配置）
- トレンド照合（6領域の判定）

## 機能3: パターン統計

全レポートから特定の項目を集計し、頻度順に表示する。

手順:
1. `.claude/artifacts/design-knowledge-index.md` を読み込む
2. 全レポートから指定項目を抽出
3. 集計して頻度順で表示

集計可能な項目:
- タイプスケール比率（どのスケールが最も多いか）
- カラーハーモニー（どのハーモニーが最も多いか）
- ナビゲーション型（どの型が最も多いか）
- 8pxグリッド準拠率（平均、分布）
- 業界分布
- トレンド照合結果（各領域のトレンド/独自/クラシック比率）

出力形式:
```
## パターン統計: {項目名}

分析済みサイト数: {N}

| {項目} | サイト数 | 割合 | 該当サイト |
|--------|---------|------|----------|
```

## 機能4: 学習進捗

分析履歴のサマリーと、5軸ごとの学び蓄積量を表示する。

手順:
1. `.claude/artifacts/design-knowledge-index.md` を読み込む
2. 分析済みサイト数、業界分布、直近の分析日を集計
3. 5軸ごとに「学びのポイント」の蓄積量を概算
4. 学習の偏りがあれば推奨する分析対象を提案

出力形式:
```
## 学習進捗レポート

### サマリー
- 分析済みサイト数: {N}（learn: {n1}, audit: {n2}）
- 業界カバレッジ: {業界一覧}
- 直近の分析: {日付} - {サイト名}

### 5軸の学び蓄積

| 軸 | 蓄積量 | 主な学び |
|---|--------|---------|
| 視覚原則 | {件数} | {代表的な学び} |
| タイポグラフィ | {件数} | {代表的な学び} |
| 配色 | {件数} | {代表的な学び} |
| スペーシング | {件数} | {代表的な学び} |
| UIパターン | {件数} | {代表的な学び} |

### 推奨
- {偏りに基づく推奨: 「EC系サイトの分析が少ないです。EC系を分析するとUIパターンの知見が広がります」等}
```

# doc-interview: Confluenceドキュメント ヒアリング & 生成

ConfluenceページのURLを受け取り、ユーザーへのヒアリングを通じて未完成ドキュメントを完成させる。

## 起動

引数としてConfluenceページのURLを受け取る。

```
/doc-interview https://your-site.atlassian.net/wiki/spaces/SPACE/pages/1973682230
```

## 処理フロー

### Step 1: ページ取得と分析

1. URLからページIDを抽出する（`/pages/(\d+)` パターン）
2. `mcp__plugin_atlassian_atlassian__getConfluencePage` でページ内容を取得する
   - cloudId: `your-site.atlassian.net`
   - contentFormat: `markdown`
3. 取得した内容を分析し、以下を特定する:
   - ページのタイトルとテーマ
   - 既存の構造（見出し、セクション）
   - TODO項目やメモ書き
   - 不足している情報

### Step 2: 既存内容の評価と方針決定

ページの完成度を判断し、ユーザーにAskUserQuestionで方針を確認する。

判断基準:
- ほぼ空（TODOのみ、メモ書きのみ）→ 「新規作成」を提案
- ある程度の内容あり → 「補完・追記」を提案
- ユーザーが最終決定する

### Step 3: ハイブリッド・ヒアリング

ドキュメント完成に必要な情報をユーザーから収集する。

ヒアリングの進め方:
1. まず、既存内容をベースに必要な質問をまとめてAskUserQuestionで提示する（2〜4問）
   - 質問はページのTODOやタイトルから自然に導出する
   - 選択肢形式で答えやすくする（自由記述のOther選択肢も常にある）
2. 回答を受けた後、情報が不足していれば追加で深掘り質問する
3. 「もう十分」「これでOK」などの意思表示があれば次のステップへ進む
4. 2〜3ラウンド程度で収束させることを目指す（質問攻めにしない）

ヒアリングのコツ:
- ユーザーが持っている暗黙知を引き出すことが目的
- 具体例や実体験を聞くと聞いた情報以上に良いドキュメントになりやすい
- 「他に追加したいことはありますか？」で締めくくる

### Step 4: ドキュメント生成

収集した情報を元に、Confluenceに適した構造のドキュメントを生成する。

生成のガイドライン:
- ページタイトルに合った内容にする
- 明確な見出し構造（h2, h3）を使う
- 箇条書きを活用して読みやすくする
- ユーザーの回答をそのまま転記するのではなく、ドキュメントとして自然な文体に整形する
- 専門用語はユーザーが使った表現をそのまま使う

### Step 5: 出力先の選択

生成したドキュメントの出力先をAskUserQuestionでユーザーに選択してもらう。

選択肢:
1. チャット上に表示 — 内容を会話内に出力する。確認後にConfluenceへの書き込みも可能
2. ローカルファイルに保存 — markdownファイルとしてローカルに保存する
3. Confluenceに直接書き込み — `mcp__plugin_atlassian_atlassian__updateConfluencePage` で直接更新する

Confluenceへの書き込み時:
- 書き込み前に最終確認を行う
- contentFormat は `markdown` を使用する
- ページタイトルは既存のものを維持する（変更が必要な場合はユーザーに確認）

## 注意事項

- 全てのコミュニケーションは日本語で行う
- ヒアリングは押し付けがましくならないよう、自然な対話を心がける
- ユーザーの回答が曖昧な場合は、具体例を挙げて確認する
- ドキュメントの品質 > 完全性。無理に全セクションを埋めるより、確かな情報で質の高い内容を書く

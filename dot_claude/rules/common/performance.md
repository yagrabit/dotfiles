# Claude Code効率的利用ルール

## モデル選択の指針

- sonnet: コードレビュー、定型的な実装、テスト生成など、高速・低コストが求められるタスク
- opus: 複雑な設計判断、アーキテクチャ分析、曖昧な要件の解釈など、高品質な推論が必要なタスク

## コンテキスト管理

- 大きなファイルは必要な部分のみ読む（Read の offset/limit を指定する）
- ファイル検索には Grep/Glob ツールを使い、Bash で grep/find を実行しない
- サブエージェントに委譲してメインコンテキストの消費を抑える
- 不要になった情報を繰り返し参照しない

## サブエージェント活用

- 独立したタスクは並列でサブエージェントに委譲する
- 調査系タスクは Explore エージェントを使い、メインの文脈を汚さない
- サブエージェントには十分なコンテキスト（ファイルパス、目的、制約）を渡す

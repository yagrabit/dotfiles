# セキュリティ共通ルール

## 秘密情報の取り扱い

- APIキー、トークン、シークレットをコードにハードコードしない
- 秘密情報ファイル(.env, *.pem, *.key, credentials.json等)の読み込みはblock-secret-filesフックで自動ブロックされる

## コーディング時の原則

- ユーザー入力は常にバリデーション・サニタイズしてから使用する
- SQLクエリには必ずパラメータバインディングを使用する（文字列結合でクエリを組み立てない）
- `innerHTML` / `dangerouslySetInnerHTML` の使用を避け、テキスト挿入やサニタイズ済みのレンダリングを使う
- `eval()`、`new Function()` の使用を禁止する
- 外部URLへのリダイレクトはホワイトリスト方式で制御し、オープンリダイレクトを防ぐ

## 依存関係の管理

- 既知の脆弱性がある依存パッケージは速やかに更新する
- `npm audit` / `pip audit` 等の警告を放置せず対応する
- 使われていない依存は削除する
